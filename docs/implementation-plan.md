# 웹 버전 구현 계획

1. `BootScene`에서 실측 프레임 규격으로 에셋을 로드하고 모든 유닛의 idle/move/attack 애니메이션을 만든다.
2. `StartScene`에서 한국어 난이도와 조작법을 제공한다.
3. `GameScene`은 3600px 전장, 고정 HUD, 드래그/휠/키보드 카메라를 조정한다.
4. `CombatUnit`과 `BaseEntity`는 화면 표현과 상태만 담당하고, 이동·전투·경제·AI는 각각 systems로 분리한다.
5. 먼저 기본 보병 생산, AI 보병, 기지 전투와 승패를 연결한다. 동일 인터페이스를 유지한 채 데이터 정의만 확장해 모든 병종을 연다.
6. 전직 UI는 원본의 순차 일반 전직과 독립 스페셜 양자택일을 유지한다. 키보드 1~8과 클릭을 모두 지원한다.
7. 공격 애니메이션 프레임 10에서 전투 콜백을 실행해 단 한 번 피해를 준다. 원거리 발사체는 해당 프레임에서 생성하되 피해 판정은 목표 도착 시 실행한다.
8. `npm run build`와 실제 브라우저 플레이를 검증하고 GitHub Actions Pages 워크플로를 제공한다.
