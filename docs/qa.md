# 검증 기록

최종 검증일: 2026-07-15

- `npm install`: 성공, 설치 시 감사 결과 취약점 0건.
- `npm run build`: TypeScript strict 검사 및 Vite 프로덕션 빌드 성공.
- 결과물: `dist/index.html`, 상대 경로 `./assets/...` 확인, 55MB.
- 에셋: `public/assets` 28개 전부 0바이트가 아니며 `dist`에 28개 모두 복사됨. JS/CSS 포함 `dist/assets` 30개.
- 투명 배경: 하늘을 제외한 27개 PNG의 알파 채널, 투명 모서리와 어두운 배경 합성 결과를 확인. 로고·유닛·기지·원경·지면의 흰 체크무늬가 제거됨.
- 프레임 정규화: 원본 1448×1086 시트의 행 전체에서 유닛 22종, 264개 프레임을 복원. 최종 496×400 셀의 발 기준점 X=204, Y=399로 정렬하고 모든 프레임의 좌우 여백·발 기준선·전체 시트 1984×1200을 재검증.
- 런타임 앵커: 모든 전투·도감·랭킹 유닛을 `setOrigin(0.5, 1)`로 통일. 고정 `frameOffsetX=44`, `frameOffsetY=0`을 적용해 상태 전환 중 sprite 위치와 표시 크기를 유지.
- 로딩 화면: 게임 캔버스보다 먼저 로고를 표시하고 배경 이미지만 이중 블러 처리. 로고와 진행률은 선명하게 유지.
- 이동 전투: 사거리 진입만으로 이동을 멈추지 않으며, 공격 애니메이션 중에도 적·기지 충돌선까지 계속 전진하도록 정적 검증.
- 돌진 병종: 일반 기병 기본 이동 1.8배, 윙드 후사르/사나다가 8칸 동안 1.8→2.4배로 가속하고 첫 유닛/기지 충돌에 최대 50% 피해를 주는지 확인. 돌진 중 속도선과 충돌 시 전용 타격 효과도 확인. 기병·로닌·사나다의 최초 공격 쿨다운은 0초, 그 외 병종은 공격 간격의 20%.
- 지면·카메라: 월드 높이 2200px, 지표면 Y=2000으로 확장하고 실제 지면 텍스처의 첫 알파 픽셀(Y=488)과 기지 하단 알파 경계를 지표면에 정렬. 화면 높이별 하단 고정 좌표를 사용해 줌 0.65~1.25에서도 지면의 화면 Y가 변하지 않도록 계산 검증.
- 개발 서버: `http://localhost:5173/` HTTP 200 및 한국어 HTML 응답 확인.
- 직접 화면 자동화: 현재 세션의 인앱 브라우저 목록이 비어 있어 클릭·스크린샷 기반 검증은 수행하지 못했다. 별도 브라우저 도구로 우회하지 않았다.
- 최종 정적 검사: `TODO`, `FIXME`, 하드코딩된 localhost 및 루트 `/assets/` 경로 없음.
